# Versioning Guidelines

When a repository uses governed versioning, keep the following in sync:
- CHANGELOG.md
- versioning/<ID>.pdf summary artifact
- versioning/<ID>.md AI/development prompt artifact

Rules:
1. Never delete historical versioning artifacts.
2. New work appends to changelog; do not rewrite old entries except factual corrections.
3. Use one unique ID per change package.
4. The markdown prompt artifact must be derivable from the PDF summary artifact.
