# Local Context Guide

Use these local governance files as the primary context for AI-assisted development work:
- .governance/controller.md
- .governance/execution_authority_and_automation_rules.md
- .governance/codingLessonsLearnt.md
- .governance/ui_ux_rules.md
- .governance/versioning-guidelines.md

## Behavior expectation
Treat user requests as execution instructions when the next step is clear.
Do not ask repeated permission for GitHub, Jira, changelog, governance, or documentation updates when those are already implied by the request.

## Recommended reading order
1. .governance/controller.md
2. .governance/execution_authority_and_automation_rules.md
3. .governance/codingLessonsLearnt.md
4. .governance/ui_ux_rules.md when UI is affected
5. CHANGELOG.md and the latest versioning artifacts
