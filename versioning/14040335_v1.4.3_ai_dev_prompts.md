# 14040335_v1.4.3 — AI fejlesztési promptok

## Cél
A Pubapp / Kapakka rendszerben a Site Admin teljesen különüljön el a venue-admin felülettől, miközben a már elkészült `common_admin` capability set változatlanul elérhető maradjon, de immár a külön siteadmin felületen.

## Kötelező szabályok
1. Először olvasd el a `codingLessonsLearnt.md` és a `changelog.md` fájlokat.
2. Semmilyen már jól működő venue-admin funkciót nem szabad elrontani.
3. A changelog csak append-only módon bővülhet.
4. A siteadmin és venue-admin shell / navigation vizuálisan és útvonal-szinten is különüljön el.
5. A `CommonAdminPanel` ne vesszen el, hanem a külön Site Admin dashboard elsődleges tartalma legyen.

## Implementációs feladatok
- hozz létre külön `src/app/siteadmin/layout.tsx` fájlt superadmin auth checkkel
- hozz létre külön `src/app/siteadmin/page.tsx` dashboardot
- a `CommonAdminPanel` ott jelenjen meg elsődlegesen
- a `/siteadmin/venues` ugyanebbe a külön layoutba essen bele
- a `src/app/admin/layout.tsx` venue-admin oldalsávjából vedd ki a keveredést okozó közvetlen Site Admin menüt
- superadminnak adj elkülönített átvezető CTA-t a venue-admin shellen belül
- a siteadmin route csak `superadmin` szerepkörrel legyen elérhető
